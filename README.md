# CA-Fire-Hot-Spots

***
### Section 1. Problem Statement
This project identifies hot-spots and cold-spots of historical California fires, revealing spatial patterns in fire frequency. <br>
***
### Section 2. Data Inputs
To identify fire hot spots and cold spots, the CAL FIRE historical fire perimeter dataset, containing perimeters dating back to 1878, is used. A California shape file from the California Department of Technology is also loaded in to define California’s boundary. <br>
***
### Section 3. Analysis Method
On the CAL FIRE geoportal site, it’s mentioned that statistical analysis on the data should be done with caution due to missing fire perimeters from loss of records or inadequate documentation. There are also some duplicate fires in the data-base or over-generalization of fire perimeters in some cases. <br>
To handle incomplete perimeters in the dataset, a buffer is first applied to the geometry data to attempt to close small gaps or overlaps and convert any non-polygon geometries in the data set into polygons. After attempting to fix as many perimeters as possible, any remaining invalid geometries are removed from the dataset. Redundant CAL FIRE perimeters are removed next, only keeping the largest perimeter out of each redundant fire. Duplicates are identified by matching fire name and year. <br>
Once the CAL FIRE data is cleaned, a grid made up of 10 by 10 kilometer square cells is created inside the California state boundary to calculate fire frequency across the state. The California boundary is obtained from a shape file from the California Department of Technology. To minimize distortion between each cell and create equal area cells, the California boundary data frame is reprojected into the California Albers projection system, which is based on Albers Equal Area Conic projection but optimized for California. The projection system units are in meters, which allows the grid cell size to be defined in meters. The grid bounds are defined for the horizontal and vertical minima and maxima values of the California state geometry. Once each grid cell has been created and inserted into a Geo Data Frame, the grid is clipped to the California boundary. Finally, the grid is visualized to validate proper implementation. <br>

Using the California grid, fire frequency is found for each grid cell by applying a spatial join between the CAL FIRE historical perimeter dataset and the California grid data frames based on whether or not polygons in the Geo Data Frames intersect. This allows for geometries that touch by boundary, corners, partial overlaps, or full containments to be joined together. Prior to doing this spatial join, the CAL FIRE data is also converted to the California Albers projection system to ensure the two spatial datasets being joined are in the same projection system. Additionally, a cell ID attribute is added to the data frame to enable an aggregate count of frequency of geometric intersection for each grid cell. After performing the spatial join, the joined data frame is aggregated using the cell ID, counting the number of times each grid cell intersects with a CAL FIRE perimeter. The outputted aggregate data frame is stored in a counts variable and given an attribute name of overlap counts. The counts aggregate data frame is then merged back with the original California grid data frame on the cell ID attribute. Any NaN values are replaced with a zero. The outputted Fire Frequency data frame is visualized.  A basemap is added to include more context as to where the data is geographically located. Note that in this visual, the data is reprojected into Web Mercator, but this reprojection is only done for the visual to be compatible with the contextily basemap function. <br>

To identify hot spots and cold spots, a Moran scatterplot is created using the fire frequency dataset. The spatial weight matrix is constructed using Queen contiguity and converted into row-standardized weights. The Global Moran’s I of the fire frequency dataset is calculated using a permutation test with 200 permutations to determine if there is a statistically significant pattern in the fire frequency data. Next, the Local Moran’s I is calculated for visualizing local spatial association strength and statistical significance. This allows for identifying significant high-high and low-low spatial clusters. The hot-spot and cold-spot map is then generated and visualized using the Local Moran’s I or the Local Indicators of Spatial Autocorrelation (LISA). <br>
Finally, an interactive visual is created in a Folium map to allow users to scroll through the map, zoom in, and click on each cell to view the total number of fires that happened within each area. Cluster labels are generated from the Local Moran’s I data set and added to the fire frequency data frame. A custom colormap is created for the Folium map. The fire frequency dataset is reprojected to the World Geodetic System 1984 for compatibility with Folium map.  The Folium map is initialized, and then the fire frequency data frame is added to the map as a GeoJSON layer where it is colored based on its cluster category. A tooltip is added to display the cluster category and fire frequency when users hover over each cell. A custom legend to display cluster categories is built in html, inserted into the map’s html template, and rendered into the outputted Folium map. <br>

### Section 4: Results and Conclusion
Interactive Folium map visuals can be viewed in the Jupyter Notebook. <br>
The calculated Moran’s I value for the dataset is about 0.658, a strong positive Moran’s I, indicating a spatial clustering pattern. The calculated p-value in the Moran’s I test is about 0.005, indicating that the clustering in the data set is significant. The output of the LISA cluster map is shown below. <br>
